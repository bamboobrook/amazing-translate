# Amazing Translate macOS Sharing Guide

This guide is for friends or testers using macOS + Chrome. The package does not include any API Key. Each user enters their own DeepSeek or GLM API Key in the extension side panel or options page.

## Recommended Flow

Create a GitHub Release and upload the ZIP generated by `npm run package`, for example:

```text
amazing-translate-0.1.0.zip
```

Then send Mac users this command:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/bamboobrook/amazing-translate/main/scripts/install-macos.sh)"
```

The script will:

- Download the latest `amazing-translate-*.zip` from GitHub Releases.
- Unpack it to `~/Applications/AmazingTranslate`.
- Open that folder in Finder.
- Open Chrome's `chrome://extensions/` page.
- Print the remaining manual steps.

Chrome does not allow scripts to silently install unpacked extensions. The user still needs to enable Developer mode, click Load unpacked, and choose the install folder.

## Manual Steps For Users

After running the script:

1. Open `chrome://extensions/` in Chrome.
2. Enable `Developer mode` in the top-right corner.
3. Click `Load unpacked`.
4. Select this folder:

```text
~/Applications/AmazingTranslate
```

5. Click the Amazing Translate icon and open the side panel.
6. Select DeepSeek or GLM, then enter your own API Key.
7. Click `Test connection`.

## Upgrade

Users can run the same command again:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/bamboobrook/amazing-translate/main/scripts/install-macos.sh)"
```

Then open `chrome://extensions/` and click reload on the Amazing Translate card.

Keep the install folder stable:

```text
~/Applications/AmazingTranslate
```

Using the same path helps Chrome keep the same local extension storage, so saved API Keys usually remain available. If users load the extension from a different path, Chrome may create a different unpacked extension ID and require API Key setup again.

## Private Repo Or Temporary ZIP URL

For a private repository, users can provide a GitHub token. The token must be used both to fetch the installer script and inside the installer when it calls the GitHub Releases API:

```bash
GITHUB_TOKEN=ghp_xxx bash -c 'curl -fsSL -H "Authorization: Bearer $GITHUB_TOKEN" https://raw.githubusercontent.com/bamboobrook/amazing-translate/main/scripts/install-macos.sh | GITHUB_TOKEN="$GITHUB_TOKEN" /bin/bash'
```

For a ZIP hosted somewhere else, provide a direct ZIP URL:

```bash
AMAZING_TRANSLATE_ZIP_URL="https://example.com/amazing-translate-0.1.0.zip" \
  /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/bamboobrook/amazing-translate/main/scripts/install-macos.sh)"
```
